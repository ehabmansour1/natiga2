# بوابة نتيجة الشهادة الإعدادية — محافظة المنيا 2025/2026

صفحة عربية (RTL) للاستعلام عن نتيجة الطلاب **برقم الجلوس** أو **بالاسم**، بتصميم بطاقة نتيجة رسمية قابلة للطباعة.

موقع ثابت بالكامل (HTML/CSS/JS) — لا يحتاج خادمًا ولا قاعدة بيانات. يقرأ ملف JSON واحدًا مُستخرَجًا من اسطوانات النتيجة (PDF).

- **عدد الطلاب:** ٨٣٬٨٢٣ (قطاع شمال + جنوب + وسط)
- **حجم البيانات:** ‎13.6MB‏ خام / ‎~2.2MB‏ بعد الضغط (gzip) — تُحمّل مرة واحدة وتُخزَّن مؤقتًا.

---

## التشغيل والنشر

- **معاينة محلية:** يجب فتح الصفحة عبر خادم HTTP (لقراءة ملف الـ JSON)، لا عبر `file://`:
  ```bash
  python -m http.server 8000
  ```
  ثم افتح `http://localhost:8000`.
- **النشر:** ارفع كل الملفات (عدا مجلد `natiga/`) على أي استضافة ثابتة:
  GitHub Pages أو Netlify أو Vercel أو Cloudflare Pages. هذه الاستضافات تضغط الملفات تلقائيًا (gzip/brotli) وتتحمّل ضغط يوم النتيجة.

> **خصوصية:** ملف `data/students.json` يحتوي فقط على (الاسم، رقم الجلوس، المدرسة، النوع، القطاع، الدرجات) — **بدون** الرقم القومي أو كود الطالب.

## تحديث البيانات (من اسطوانات الـ PDF)

1. ضع ملفات الـ PDF داخل مجلد `natiga/` بالأسماء الموجودة في `tools/parse_natiga.py` (قائمة `SECTORS`).
2. ثبّت المكتبة وشغّل المحوّل:
   ```bash
   pip install pymupdf
   python tools/parse_natiga.py
   ```
3. يُنشئ هذا الأمر `data/students.json` تلقائيًا، فيلتقطه الموقع مباشرة.

أوامر مساعدة:
```bash
python tools/parse_natiga.py --validate     # فحص + عيّنات بدون كتابة الملف
python tools/parse_natiga.py --pages=15      # حدّ صفحات لكل ملف (اختبار سريع)
python tools/parse_natiga.py --only=وسط      # قطاع واحد فقط
```

### كيف يعمل المحوّل
- يستخرج كل صف طالب اعتمادًا على ثوابت لا تتغيّر بين التخطيطات الثلاثة:
  - عمود **النوع** (ذكر/أنثى) يقع دائمًا يسار الدرجات (نقطة ارتساء).
  - **المجموع الكلي = مجموع المواد الست** (تحقّق ذاتي لكل صف).
- يُصلح مشاكل ترميز الخط العربي قدر الإمكان: `محمد`، `الله`، حرف الياء، التطويل.

### حدود معروفة في استخراج الأسماء
ترميز خطوط الـ PDF يسبب أخطاء حروف بسيطة في **بعض** الأسماء (ليغاتورة لام-ألف مثل «أحلام»، وحرف القاف في قطاع جنوب يظهر لامًا). لذلك:
- **البحث برقم الجلوس دقيق 100%** وهو الأساس.
- **البحث بالاسم تقريبي ومتسامح** (يوحّد الأشكال المتشابهة) وقد يُظهر عدة نتائج للاختيار منها.

## التخصيص (في `app.js` أعلى الملف ضمن `CONFIG`)

| الإعداد | الوظيفة |
|---|---|
| `authority` / `examTitle` / `year` | نصوص ترويسة بطاقة النتيجة |
| `dataPath` | مسار ملف البيانات (افتراضي `data/students.json`) |

ويمكن تعديل **الدرجات الكبرى/الصغرى** ومنطق النجاح في `MAIN_SUBJECTS` و `EXTRA_SUBJECTS` و `computeResult`.
